 **Hash:**
		A unique fixed length string, meant to identify a piece of data, they are created by placing said data inside a hash function
- Ethereum uses Keccak-256 hashing algorithm 
- ![[Pasted image 20260512133413.png]]
- the length of the string remains constant 
- each unique input data will have a unique hash
**BLOCK**
- A block hashes 3 fields:
	- the block 
	- Nonce
	- Data
- into a single hash
- ![[Pasted image 20260512133709.png]]
- Here the miner solved the problem to find the nonce such that for block no. 1 and given data the hash starts with 0000.
**BLOCKCHAIN**
- ![[Pasted image 20260512133944.png]]
- in a block chain along with block no., data, nonce, the hash of previous block is also taken into calculation
- The beginning block of the block chain is know as the **The Genesis Block**
- thus whilst mining not only we have to find a nonce that satisfies the given block but also that satisfies the hash of the given block and in cascade all the linked blocks hence making the computation more complex
- ![[Pasted image 20260512134321.png]]
- any change invalidates the block hence the entire chain requiring to mine to validate the change
**DISTRIBUTED BLOCK CHAIN**
![[Pasted image 20260512134514.png]]
- here multiple entities have the copy of the block chain name peer 1 and peer 2 
- thus change made in a single block chain needs to be validated in every copy with every user
- without validation a peer 1 will have his own version of block chain this the chain is said to be forked
